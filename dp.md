# Dynamic Programming Roadmap

A structured plan to master DP patterns for coding interviews.

---

# How to Use This

For every pattern:

1. Learn the first problem from YouTube or article.
2. Implement it yourself.
3. Solve the remaining problems without help.

Time rule:

* Try 30 minutes
* If stuck → learn solution
* Recode without watching

---

# Pattern 1: 0/1 Knapsack

Concept: Choose or skip items.

Core Problem:

* 0/1 Knapsack

Practice:

* Partition Equal Subset Sum
* Target Sum
* Last Stone Weight II

Goal:
Understand subset-style DP.

---

# Pattern 2: Unbounded Knapsack

Items can be used multiple times.

Learn:

* Coin Change

Practice:

* Coin Change II
* Combination Sum IV
* Perfect Squares

---

# Pattern 3: LIS Pattern

Increasing subsequence problems.

Learn:

* Longest Increasing Subsequence

Practice:

* Number of LIS
* Russian Doll Envelopes

---

# Pattern 4: String DP

DP on two strings.

Learn:

* Longest Common Subsequence

Practice:

* Edit Distance
* Delete Operation for Two Strings
* Longest Palindromic Subsequence

---

# Pattern 5: Grid DP

Movement in a grid.

Learn:

* Unique Paths

Practice:

* Minimum Path Sum
* Unique Paths II
* Maximal Square

---

# Pattern 6: Kadane / Subarray

Learn:

* Maximum Subarray

Practice:

* Maximum Product Subarray
* Maximum Sum Circular Subarray

---

# Pattern 7: House Robber

Linear DP decisions.

Learn:

* House Robber

* House Robber II
* Delete and Earn

# Pattern 8: Interval DP

# Key DP Thinking Process

1. Define state
2. Find choices
3. Write recurrence
4. Memoization
5. Convert to tabulation

---

# Common DP State Types

Index
Index + Sum
Index + Capacity
Index + Previous
Two strings (i,j)
Grid (row,col)

---

# Signals to Recognize Patterns

Take / Skip → Knapsack
Increasing sequence → LIS
Two strings → LCS style
Grid movement → Grid DP
Subarray best value → Kadane

---

# Target Goal

Solve ~60 DP problems.
After that patterns repeat and become easy.

---

# Tips

* Write recursion first
* Add memoization
* Convert to tabulation
* Practice writing states clearly

---

# Final Advice

Do not memorize solutions.
Recognize patterns.

Consistency > number of problems.
